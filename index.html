Learn DSA: Dijkstra's Algorithm

🎯 What is Dijkstra's Algorithm?

Dijkstra's algorithm is a greedy algorithm that finds the shortest path between nodes in a weighted graph. It was conceived by computer scientist Edsger W. Dijkstra in 1956.

Key Characteristics:
  • ✅ Greedy approach: Always picks the closest unvisited node
  • ✅ Optimal solution: Guarantees shortest path (with non-negative weights)
  • ✅ Time efficient: O((V + E) log V) with min-heap
  • ❌ Limitation: Cannot handle negative edge weights

When to Use Dijkstra's?

  • 🗺️ GPS Navigation: Finding shortest route (Google Maps, Waze)
  • 🌐 Network Routing: OSPF protocol for internet routing
  • 🎮 Game Development: AI pathfinding for NPCs
  • 📱 Social Networks: Recommendation systems
  • 📡 Telecommunications: Optimal data routing

📝 Dijkstra's Algorithm Pseudocode

function dijkstra(Graph, source): dist[source] ← 0 for each vertex v in Graph: if v ≠ source: dist[v] ← INFINITY unvisited ← all vertices pq ← PriorityQueue() // min-heap based on distance pq.insert(source, 0) while pq is not empty: u ← pq.extractMin() if u is visited: continue mark u as visited for each edge (u, v) with weight w: if dist[u] + w < dist[v]: dist[v] ← dist[u] + w pq.insert(v, dist[v]) parent[v] ← u // for path reconstruction return dist[], parent[]

Key Variables Explained:

  • dist[]: Stores the shortest distance found from source to each vertex
  • pq (Priority Queue): Efficiently selects the minimum distance vertex
  • visited: Track which vertices have been processed
  • parent[]: Used to reconstruct the actual path (optional)
  • Relaxation: The process of updating shorter distances

👣 How Dijkstra's Algorithm Works: Step-by-Step

Example: Finding shortest path from "Mirpur 14" to "Hatirjheel"

1Initialize: Set distance to source = 0, all others = ∞

Mirpur 14: 0
Mirpur 10: ∞
Banani: ∞
... (all others: ∞)

2Select Minimum: Pick unvisited node with smallest distance (Mirpur 14 = 0)

3Explore Neighbors: For each neighbor of Mirpur 14:

  • Check if: current distance + edge weight < neighbor's distance
  • If yes, update the neighbor's distance
  • Add neighbor to priority queue

4Repeat: Pick next minimum distance node and repeat until destination is reached or all nodes visited

💡 Why It Works:

Once a node is visited, its distance is final. Visiting nodes in order of distance guarantees we find the shortest path first.

⚡ Complexity Analysis

Metric Value Explanation
Time Complexity O((V + E) log V) With binary heap: Extract-min V times, Insert/Update E times
Space Complexity O(V) Distance array, visited set, parent array
Data Structure Priority Queue Binary heap for efficient min operations
Graph Representation Adjacency List More efficient for sparse graphs

Operation Counts for This Graph:

Why Binary Heap?

  • Extract minimum: O(log V)
  • Insert/Update: O(log V)
  • Fibonacci Heap could give O(E + V log V) but complex to implement

⚔️ Algorithm Comparison

Dijkstra's Algorithm

When: Weighted graphs, positive weights
Time: O((V+E) log V)
Space: O(V)
Optimal: ✅ YES
Greedy: ✅ YES

Bellman-Ford

When: Negative weights allowed
Time: O(V × E)
Space: O(V)
Optimal: ✅ YES
Detects: ✅ Negative cycles

BFS (Unweighted)

When: Unweighted graphs
Time: O(V + E)
Space: O(V)
Optimal: ✅ YES (for unweighted)
Simpler: ✅ YES

DFS

When: Graph traversal, connectivity
Time: O(V + E)
Space: O(V)
Shortest Path: ❌ NO
Use Case: 🎯 Different purpose

📚 Learning Resources & Tips

Recommended Learning Path:

  1. Graph Basics: Vertices, edges, weighted/unweighted
  2. Data Structures: Priority Queue, Binary Heap
  3. BFS/DFS: Basic graph traversal first
  4. Dijkstra's: Understand the greedy approach
  5. Advanced: Bellman-Ford, Floyd-Warshall
🎓 Practice Exercises:
  • Trace Dijkstra's by hand on a small graph (5-6 nodes)
  • Implement it from scratch without looking at code
  • Add path reconstruction to find actual route
  • Compare with BFS on an unweighted version
  • Test with disconnected graphs
  • Modify to find shortest path to a specific node only

Common Mistakes to Avoid:

  • ❌ Using Dijkstra's with negative weights (use Bellman-Ford)
  • ❌ Not using priority queue (gives O(V²) instead of O((V+E) log V))
  • ❌ Updating distances after marking as visited
  • ❌ Not initializing source distance to 0
  • ❌ Forgetting to track visited nodes

Implementation Tips:
